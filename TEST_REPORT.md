## 구아바 프로젝트 기능별 통합 테스트 결과서

본 문서는 프론트엔드(React) 기준 통합 테스트(E2E) 결과를 요약합니다. 테스트는 Playwright 기반으로, 백엔드 의존도를 낮추기 위해 주요 API를 테스트 내에서 스텁(모의 응답) 처리했습니다. 실제 백엔드 환경에서의 실측은 별도 환경 설정 후 재실행이 필요합니다.

- 테스트 러너: Playwright
- 테스트 위치: `frontend/tests/*.spec.ts`
- 베이스 URL: `frontend/playwright.config.ts` 설정값 사용
- 네트워크: 주요 엔드포인트 스텁 사용

### 실행 방법
```bash
cd frontend
npx playwright test
```

필요 시 UI 모드로 실행:
```bash
cd frontend
npx playwright test --ui
```

### 시나리오별 결과 요약

- 인증/마이페이지
  - 로그인 성공 시 마이페이지로 리다이렉트: PASS (스텁)
  - 마이페이지 구독 비어있음 안내 노출: PASS (스텁)
- 서비스 검색/상세
  - 검색 결과 카드 렌더링: PASS (스텁)
  - 카드 → 상세 페이지 이동: PASS (스텁)
  - 상세 플랜 목록(2종) 렌더링: PASS (스텁)
- 구독 추가 플로우
  - 사전 로그인 상태에서 플랜 선택 → 상세 입력 → 추가 완료 토스트 노출: PASS (스텁)

### 커버리지 맵(주요 기능)

- 인증
  - 로그인: 자동화 테스트로 검증(auth-login-mypage.stub.spec.ts)
  - 로그아웃: 헤더 버튼 존재 및 훅 동작은 수동 확인 권장
- 마이페이지
  - 구독 리스트/합계 표시: 빈 케이스 PASS (스텁)
  - 즐겨찾기/메모: API 스텁으로 빈 상태 PASS, 상세 기능은 백엔드 연동 환경에서 재권장
  - 카테고리별 통계: 빈 상태에서 비노출 확인(간접 검증)
  - 리포트 CSV 내보내기: 버튼 존재(동작은 백엔드 미연동 상태에서 스킵)
- 서비스 검색/목록
  - 검색 결과 표시/카드 요소 노출: PASS (스텁)
  - 정렬/필터 UI: 표시 검증, 필터링 정확성은 별도 유닛/통합 확장 권장
  - 즐겨찾기 토글: 인증/서버 의존 → 환경 연결 후 권장
- 서비스 상세
  - 플랜 렌더링: PASS (스텁)
  - 즐겨찾기/메모: 인증/서버 의존 → 환경 연결 후 권장
  - 구독 추가 모달: 선택/세부/추가 완료 토스트 PASS (스텁)

### 비고

- 본 테스트는 프론트엔드만 수정한다는 제약을 준수합니다.
- 스텁 기반 검증으로 UI·흐름의 안정성을 보장하되, 실제 백엔드와의 통합 결과는 환경 준비 후 재측정이 필요합니다.
- 필요 시 스텁 데이터를 확장하여 경계/에러 케이스(네트워크 오류, 401/403, 빈/대량 데이터 등) 커버 가능.
